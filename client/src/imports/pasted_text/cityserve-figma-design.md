Design a complete, fully functional UI/UX system in Figma for a web application called:

"CityServe – Local Service Booking Platform"

This is a role-based platform with three user types:
1. User (Customer)
2. Service Provider
3. Admin

The design must include full user flows, interactive components, and modern UI/UX principles. The output should be structured as a real product design system with reusable components, consistent spacing, and responsive layouts.

--------------------------------------------------

🎨 DESIGN STYLE & SYSTEM

- Style: Modern, minimal, clean, professional
- Color palette:
  - Primary: Blue (#2563eb)
  - Background: Light gray (#f8fafc)
  - Cards: White
  - Text: Dark gray
- Use:
  - Glassmorphism for modals and overlays
  - Soft shadows and elevation for depth
  - Rounded corners (12px–16px)
  - Clean typography (Inter or similar)
- Layout:
  - Grid-based responsive layout
  - Proper spacing system (8px scale)
- Components:
  - Buttons, Inputs, Cards, Modals, Tables, Badges
  - Navbar and Sidebar navigation

--------------------------------------------------

👤 USER DASHBOARD (CUSTOMER UI)

Design the following screens:

1. Authentication Pages
   - Login / Signup (centered card layout)

2. User Dashboard
   - Location selector dropdown
   - Category cards (Salon, Home Services, etc.)
   - “Your Bookings” section

3. Service Listing Page
   - Grid layout of service provider cards
   - Each card: name, service, price, rating, "Book Now"

4. Booking Page
   - Service details
   - Date picker
   - Time slot selection (Morning / Afternoon / Evening)

5. Booking Status Page
   - Cards showing:
     - Pending
     - Accepted
     - In Progress
     - Completed
     - Rejected
   - Show PIN when booking is accepted

6. PIN Display UI
   - Clean OTP-style input boxes (4–6 digits)

7. Ratings & Favorites UI

-----------------------------------

✨ USER DASHBOARD EFFECTS

- Skeleton Screens (loading states for lists/cards)
- Staggered Reveal Animations (service cards load smoothly)
- Pulse Effect (booking confirmation and alerts)
- Haptic Feedback (for mobile interactions like booking)
- Glassmorphism (modals like booking confirmation)
- Micro-interactions:
  - Button click success animation
  - Hover + press states

--------------------------------------------------

🧑‍🔧 SERVICE PROVIDER DASHBOARD

Design:

1. Provider Dashboard
   - Stats cards (Total, Completed, Pending bookings)

2. Booking Management
   - Table/list view
   - Accept / Reject buttons

3. Schedule View
   - Daily bookings

4. PIN Entry Screen
   - Input for service completion code

5. Service Management Page
   - Add/Edit services form

-----------------------------------

✨ PROVIDER UI EFFECTS

- Elevation & Soft Shadows (cards with depth)
- Slide-in Drawers (side panels for editing services)
- Hover States (calendar/time slot selection)
- Inline Validation:
  - Form errors and success states
- Active Indicators:
  - Real-time booking status highlights
- Parallax Scrolling:
  - Smooth navigation in schedule view

--------------------------------------------------

🛠️ ADMIN DASHBOARD

Design:

1. Admin Dashboard Overview
   - Stats (users, providers, bookings)

2. Provider Approval Page
   - Table with approve/reject actions

3. Location Management
   - Approve new city requests

4. Category Management
   - Approve new service categories

5. Booking Monitoring

-----------------------------------

✨ ADMIN UI EFFECTS

- Easing Curves (smooth transitions in charts)
- Toast Notifications:
  - Success / Error messages
- Background Blur:
  - Focus on modals
- Color-coded Status Pulsing:
  - Alerts (pending approvals, issues)
- Tooltip Fade-in:
  - Data explanations
- Ripple Effect:
  - Button click feedback

--------------------------------------------------

🔄 UX FLOW REQUIREMENTS

Ensure full flows are designed:

USER FLOW:
Login → Select Location → Browse Services → Book → Track Status → PIN → Complete → Review

PROVIDER FLOW:
Login → Accept Booking → Perform Service → Enter PIN → Complete

ADMIN FLOW:
Login → Approve Providers → Manage Locations → Monitor System
